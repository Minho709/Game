# 데스크 클리어 (Desk Clear) - 게임 기획안

## 핵심 컨셉

어질러진 가상 책상 위 아이템들을 제한 시간 내에 올바른 자리로 정리하는 퍼즐 게임.
작업 중 막히는 순간, 잠깐 뇌를 환기시키기 위한 3~10분짜리 브레인 브레이크 게임.

---

## 기술 스택

| 역할 | 기술 |
|------|------|
| 게임 클라이언트 | HTML / CSS / Vanilla JS |
| 배포 | GitHub Pages |
| 스코어보드 DB | Supabase (PostgreSQL) |

---

## 핵심 UX 원칙

- 3초 안에 규칙 이해 가능
- 실수해도 짜증 안 남 (패널티 가볍게)
- 한 판 후 바로 재시작 가능
- 정리하면서 개운한 게임 (빡치는 게임 아님)

---

## 게임 플로우

```
닉네임 입력
→ 레벨 1 고정 시작 (튜토리얼 없이도 이해 가능한 구조)
→ 랜덤 배치된 책상 등장
→ 아이템 정리 (방해 요소는 2판째부터 등장)
→ 완료 or 시간 초과
→ 점수 계산
→ Supabase 자동 저장
→ 스코어보드 확인
→ 다음 레벨 or 재시작
```

---

## 세션 구조

**누적형**: 실패할 때까지 레벨이 계속 올라가며 점수 누적
- 클리어 시 자동으로 다음 레벨 진행
- 시간 초과 시 게임 종료 → 최종 점수 저장
- 언제든 중단 가능 (중단 시 현재 점수 저장)

---

## 조작 방식

| 구분 | 방식 |
|------|------|
| 기본 조작 | 클릭 후 선택 (아이템 클릭 → 구역 클릭) |
| 보조 조작 | 드래그앤드롭 지원 |

**개발 순서**: 클릭 방식 먼저 완성 → 드래그앤드롭 추가

### 조작 피드백
- 선택된 아이템: 테두리 + 확대 + 그림자 강조
- 올바른 구역 클릭: 자동 스냅 (정리 쾌감의 핵심)
- 잘못된 구역 클릭: 원위치 복귀 + 가벼운 흔들림
- 배치 완료: 효과음 + 정돈되는 애니메이션

---

## 아이템 종류 및 정리 구역

| 아이템 | 정리 구역 |
|--------|----------|
| 📄 문서/파일 | 파일함 |
| ☕ 커피컵 | 선반 |
| 📝 포스트잇 | 메모판 |
| 🖊️ 펜/연필 | 펜꽂이 |
| 📚 책 | 책장 |

### 정리 구역 판정 규칙
- 구역 영역 안에 클릭 또는 드롭하면 자동 스냅
- 정확한 좌표 불필요, 구역 안이면 인정
- 드롭 전 올바른 구역 위에 올리면 하이라이트 표시 (미리보기)
- 잘못된 구역은 하이라이트 없음 → 시각적으로 구분 가능

---

## 랜덤성 시스템

- 매판 아이템 위치 랜덤 배치 (패턴 외우기 불가)
- 전체 아이템 풀에서 레벨에 맞는 수량만 랜덤 선택
- 레벨 3 이상부터 정리 구역 위치도 매판 조금씩 변경

---

## 방해 요소 시스템

| 방해 요소 | 등장 조건 | 효과 |
|----------|----------|------|
| 📨 긴급 메일 | 레벨 2부터, 30초마다 1회 | 새 아이템 1개 추가 투하 |
| 💨 바람 | 레벨 3부터 | 아이템 1개 랜덤 이동 |
| ⚡ 우선순위 아이템 | 레벨 2부터 | 특정 아이템에 타이머 부착, 먼저 처리 안하면 감점 |
| 🔀 구역 변경 | 레벨 4부터 | 정리 구역 하나 일시적 이동 |

### 방해 강도 제한 원칙
- 한 번에 하나의 방해만 발생 (중첩 없음)
- 방해 발생 전 시각/사운드 예고 (0.5초)
- 레벨이 낮을수록 방해 빈도 낮음
- 방해 요소는 스트레스가 아닌 변화로 느껴지는 수준 유지

---

## 난이도 시스템

| 레벨 | 아이템 수 | 시간 | 카테고리 수 | 방해 요소 |
|------|---------|------|------------|---------|
| 1 | 6개 | 50초 | 2가지 | 없음 |
| 2 | 9개 | 55초 | 3가지 | 우선순위 아이템, 긴급 메일 |
| 3 | 12개 | 55초 | 4가지 | + 바람 |
| 4 | 15개 | 50초 | 4가지 | + 구역 변경 |
| 5 | 18개 | 50초 | 5가지 | 전체 |
| 6+ | +2개씩 | 45초 | 5가지 | 전체, 빈도 소폭 증가 |

- 한 판 기준 30~60초 내 완료 설계
- 결과 화면 포함 전체 체감 시간 90초 이내

---

## 점수 시스템

| 항목 | 점수 |
|------|------|
| 아이템 정리 1개 | +10점 |
| 시간 보너스 | 남은 시간 × 2점 |
| 콤보 (연속 3개 정확히 정리) | 해당 구간 점수 ×1.5 |
| 우선순위 아이템 시간 초과 | -20점 |

### 밸런스 기준
- 레벨 1 만점 기준 약 200점 (6개×10 + 50초×2)
- 콤보는 3개 단위로만 적용, 전체 점수에 과도한 영향 없음
- 시간 보너스가 기본 점수의 2배를 넘지 않도록 설계

## 스코어보드 (Supabase)

### DB 테이블 구조

```sql
CREATE TABLE scores (
  id         SERIAL PRIMARY KEY,
  nickname   VARCHAR(20) NOT NULL,
  score      INTEGER NOT NULL CHECK (score >= 0 AND score <= 99999),
  level      INTEGER NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 동작 방식
1. 게임 시작 전 닉네임 입력 (최대 20자, 특수문자 제한)
2. 게임 종료 시 자동으로 Supabase에 점수 저장
3. 결과 화면에서 전체 TOP 10 스코어보드 표시
4. 내 점수 순위 하이라이트

```
🏆 TOP 10
──────────────────
1위  닉네임A   3,200점
2위  닉네임B   2,850점
3위  👉 나      2,700점
...
```

### 보안 / 운영 규칙
- Supabase RLS (Row Level Security) 활성화
  - INSERT: 닉네임, 점수 범위 검증 후 허용
  - SELECT: 전체 공개
  - UPDATE / DELETE: 비허용
- 점수 범위 서버 측 CHECK 제약 (0 ~ 99,999)
- 닉네임: 최대 20자, 영문/한글/숫자만 허용, 욕설 필터 기본 적용
- 닉네임 중복 허용 (동명이인 가능, 계정 개념 없음)
- 오프라인 fallback: 인터넷 미연결 시 게임 정상 작동, 점수 저장만 스킵 + 안내 메시지

---

## UX / 연출 방향

- 배치 완료 효과음: 가볍고 청량한 클릭음
- 콤보 달성: 짧은 반짝임 효과
- 방해 요소 예고: 아이콘 + 짧은 사운드
- 전체 톤: 자극적이지 않고 깔끔한 오피스 감성
- 배경: 단순한 책상 이미지 또는 일러스트

---

## 개발 단계 (MVP 우선)

### 1단계 MVP
- 랜덤 배치 + 클릭 후 선택 조작
- 기본 정리 + 점수 계산
- 결과 화면
- 레벨 1~3

### 2단계
- 방해 요소 추가
- 드래그앤드롭 보조 입력
- 레벨 4+

### 3단계
- GitHub Pages 배포
- Supabase 연결
- 스코어보드 UI
- 보안 설정 (RLS)

### 4단계 (선택)
- 사운드 / 애니메이션 보강
- 테마 변형 (마감 전날 책상 등)
- 모바일 터치 지원
